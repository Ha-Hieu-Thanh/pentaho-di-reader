# Pentaho DI Reader

Extract SQL queries from Pentaho DI `.ktr` (Transformation) and `.kjb` (Job) XML files and generate a usage report in Excel.

## Prerequisites

- **Java 8 or higher** — tested with Liberica JDK 17
- **Maven 3.6+**
- **Windows** — the build commands below use Windows paths; adjust accordingly for Linux/macOS

## Setup JAVA_HOME

Before running Maven, set `JAVA_HOME`:

```cmd
set "JAVA_HOME=C:\Program Files\BellSoft\LibericaJDK-17"
set PATH=%JAVA_HOME%\bin;%PATH%
```

Add these to your system environment variables for permanence, or place them in a `set_env.bat` script.

## Quick Start

### 1. Compile

```cmd
cd C:\Users\Thanh\Workspace\pentaho-di-reader
mvn clean compile -Dmaven.resolver.transport.wagon
```

### 2. Run tests

```cmd
mvn test -Dmaven.resolver.transport.wagon
```

Expected output:
```
Tests run: 25, Failures: 0, Errors: 0, Skipped: 0
BUILD SUCCESS
```

### 3. Package

```cmd
mvn package -Dmaven.resolver.transport.wagon -DskipTests
```

This produces:
- `target/pentaho-di-reader-1.0.0.jar` — the application JAR
- `target/lib/` — all dependency JARs

### 4. Run

Pass a **folder path** (recursively scans all `.ktr`/`.kjb` files) or a **single file path**:

```cmd
mvn exec:java -Dexec.mainClass=com.app.Main ^
  -Dexec.args="C:\Users\Thanh\Workspace\pentaho-di-reader\sample-files" ^
  -Dmaven.resolver.transport.wagon
```

Or with the packaged JAR:

```cmd
java -cp "target\classes;target\lib\*" com.app.Main C:\path\to\your\folder
```

## Usage

```
Enter a .ktr or .kjb file path, or a folder path: C:\data\pentaho\etl
```

The tool will:
1. Recursively find all `.ktr` and `.kjb` files
2. Parse each XML file and extract SQL queries
3. Analyze each SQL statement for tables and columns used
4. Generate `sql_usage_report.xlsx` in the same directory as the input

## Output: `sql_usage_report.xlsx`

| Sheet | Description |
|---|---|
| **Summary** | All unique tables and how many times each appears across all files |
| **Table Detail** | All unique (table, column) pairs and usage count |
| **File Detail** | Each processed file with its list of tables and columns |

## Troubleshooting

### `JAVA_HOME` not found
```
Error: JAVA_HOME environment variable is not defined correctly
```
Ensure `JAVA_HOME` is set to the JDK root directory before running Maven.

### SSL / PKIX certificate errors
```
Could not transfer artifact ... PKIX path building failed
```
Use the `-Dmaven.resolver.transport.wagon` flag, or configure your corporate proxy/SSL settings in `conf/settings.xml`.

### No SQL found in report
The `.ktr`/`.kjb` files must contain SQL inside XML elements such as `<sql>`, `<query>`, `<SQL>`, or `<stepexecutioninformation>`. SQL strings are expected inside CDATA sections or as text nodes.
